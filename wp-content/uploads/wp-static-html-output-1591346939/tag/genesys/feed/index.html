I find that Genesys algorithm to be smart and most appropriate.

How it is calculated?

Please find the details below (from Stat Server deployment guide)

Estimated Wait Time (EWT) =Â  AHT * ( CIQU / (AA * EP))

where

  • AHT (Average Handle Time) is calculated as TotalTime(Mask1, Interval)/TotalNumber(Mask2, Interval)
    • Mask1 includes ACW time i.e. CallReleased, ACWCompleted, CallMissed, ACWMissed events are used for calculation.
    • Mask2 is for call i.e.) CallReleased and CallMissed events only
    • If no calls are handled by agent for this queue, AHT is assumed to be 90 seconds
  • CIQU (Calls in Queue Unassigned) â€“ Calls waiting in queue (CIQ) waiting for agent
    • CIQU = 0 if agents in ready state (AR) > CIQ
    • CIQU = CIQ if no agents are currently ready
    • CIQU = CIQ â€“ AR if some agents are in ready state
  • AA (Active Agents) â€“ Agent logged in and is busy or not ready to take calls. If AA = 0, 0.0001 is used
  • EP (Effective Portion) â€“ Total time spent by agent to process calls from the queue divided by total time spent by agent to process all calls. Default value : 1

From wiki

A leap second is a one-second adjustment that is occasionally applied to Coordinated Universal Time (UTC) in order to keep its time of day close to the mean solar time, or UT1. Without such a correction, time reckoned by Earth’s rotation drifts away from atomic time because of irregularities in the Earth’s rate of rotation.
